Panduan Lengkap Testing Lokal (Dari 0)

Panduan ini dirancang khusus dari nol bagi sistem operasi Windows. Kita akan memulai dari instalasi Docker Desktop yang benar, menyiapkan environment, hingga menjalankan aplikasi LENTERA Chatbot di laptop Anda.

Langkah 1: Install Docker Desktop (Dari 0)

Karena kita menggunakan arsitektur container, laptop Anda membutuhkan perantara bernama "Docker" untuk bisa menjalankan image database, API backend, dan frontend layaknya sebuah server.

A. Mengaktifkan WSL 2 (Windows Subsystem for Linux)

Agar Docker lebih stabil dan cepat di Windows, ia membutuhkan fitur WSL 2 diaktifkan.

  1. Klik tombol Start Windows, cari PowerShell.
  2. Penting: Klik Kanan PowerShell → Run as Administrator.
  3. Ketik perintah ini di PowerShell lalu tekan Enter:
PowerShell (Admin)
wsl --install

Jika Anda disuruh me-restart komputer/laptop setelah proses selesai, silakan Restart terlebih dahulu sebelum lanjut.

B. Download dan Install Docker Desktop

  1. Buka browser, kunjungi: Download Docker Desktop
  2. Klik tombol biru Download for Windows.
  3. Setelah selesai didownload, buka file installernya (Docker Desktop Installer.exe).
  4. Saat mode instalasi, pastikan opsi "Use WSL 2 instead of Hyper-V" sudah tercentang, lalu tekan OK dan ikuti sampai selesai (mungkin akan meminta Restart komputer lagi).

C. Jalankan Docker Pertama Kali

  1. Cari Docker Desktop di menu Start Windows lalu buka.
  2. Saat pertama dibuka, jika muncul pop-up Terms & Conditions, klik Accept.
  3. Klik tombol Continue without signing in (Anda tidak wajib membuat akun).
  4. Tunggu beberapa saat sampai indikator di pojok kiri bawah aplikasinya berwarna Hijau (bertuliskan Engine running). Jangan di-close, cukup Minimize (simpan) aplikasinya.
🔥 Validasi Sukses: Buka "Command Prompt", lalu ketik docker --version dan enter. Jika muncul tulisan seperti Docker version 24.x.x maka Docker berhasil diinstall.

Langkah 2: Siapkan Kredensial AI (GCP)

Agar backend bisa "berpikir" tanpa error, Anda wajib memasukkan kunci rahasia milik Google Cloud.

  1. Pastikan file Master Anda yang bernama gcp-key.json sudah diletakkan ke dalam folder:

d:\KADA\bpt-chatbot\credentials\gcp-key.json

Langkah 3: Konfigurasi File Environment Lokal (.env)

Di VPS kita memakai NGINX dan HTTPS. Di laptop, kita tidak punya itu. Maka kita harus "memutar" jalurnya sedekat mungkin ke koneksi lokal (localhost).

A. Password Utama (Root Env)

Buka folder d:\KADA\bpt-chatbot\. Cari file bernama .env. Pastikan isinya ada password:

d:\KADA\bpt-chatbot\.env
MONGO_ROOT_PASSWORD=PasswordKuat123
REDIS_PASSWORD=RahasiaRedis456

B. Setting Backend

Buka file bpt-komdigi-chatbot-BE/.env.production dan rubah menjadi seperti ini:

d:\KADA\bpt-chatbot\bpt-komdigi-chatbot-BE\.env.production
# Pastikan menggunakan password yang sama persis seperti poin A di atas
MONGODB_URI=mongodb://bpt_admin:PasswordKuat123@mongodb:27017/bpt_db?authSource=admin
REDIS_URL=redis://:RahasiaRedis456@redis:6379/0

# ⭐ PENTING: Karena ini pakai laptop lokal, kita ubah Domain CORS-nya ke Localhost:3000
ALLOWED_ORIGINS=http://localhost:3000

C. Setting Frontend

Buka file bpt-komdigi-chatbot/.env.production. Ganti bagian ini:

d:\KADA\bpt-chatbot\bpt-komdigi-chatbot\.env.production
# Internal (jangan dirubah) - Next.js cari backend pake nama docker "backend"
BACKEND_API_URL=http://backend:8000

# ⭐ PENTING: Untuk testing di browser laptop lokal (Client-Side), paksa panggil API melalui localhost 8000
NEXT_PUBLIC_BACKEND_API_URL=http://localhost:8000

# Pastikan sama isinya seperti Backend bagian B tadi!
MONGODB_URI=mongodb://bpt_admin:PasswordKuat123@mongodb:27017/bpt_db?authSource=admin
JWT_SECRET=sembarang_teks_bebas

Langkah 4: Jalankan Komplet via Terminal

Sekarang tahap yang sesungguhnya.

  1. Buka Command Prompt (cmd) biasa.
  2. Pindah ke folder project chatbot dengan mengetik:
CMD
cd d:\KADA\bpt-chatbot
d:
  1. Jalankan kode rahasianya untuk membangun keseluruhannya sekaligus (harap sabar menunggunya):
CMD / PowerShell
docker compose up -d --build
Sistem akan men-download database (Mongo, Redis), meng-install library Python, mendownload Model AI Embeddings (~1.5GB) dan menyusun ulang Javascript. Proses ini bisa membutuhkan 5 hingga 15 menit. Cukup lihat hingga semua proses selesai bertuliskan Started / Done tanpa error.

Langkah 5: Membuat Akses Admin Pertama

Setelah semuanya nyala dengan sukses. Anda harus menyuruh Frontend (yang sedang jalan di Docker) membuat akun admin pertama kalinya agar Anda bisa login di browser.

  1. Buka Command Prompt yang sama. Pastikan sedang berada di direktori project lalu copy paste ini lalu tekan Enter:
CMD / PowerShell
docker exec -it bpt_frontend sh -c "node scripts/seedAdmin.mjs"

Sebuah tulisan Admin 'admin' berhasil ditambahkan! akan tampil jika sukses.

Langkah 6: Buka Aplikasi & Lakukan Pengecekan

Sekarang sistemnya sudah berdiri utuh. Coba buka Browser Google Chrome / Edge untuk testing:

  • 🗣️ UI Chatbot untuk Publik: Buka situs http://localhost:3000 dan cobalah mengirim pesan bot.
  • 🔐 Buka Panel Admin CMS: Buka situs http://localhost:3000/admin. (Gunakan Username admin dan password: Admin@123).
  • 📄 Cobalah Mengelola Info: Coba fitur Upload file PDF/Docx, lalu cek di tabel manage file. Setelah berhasil terupload, cobalah tanyakan kembali informasinya langsung ke chatbot.
🔥 PENTING SAAT AKAN DEPLOY ASLI DI VPS :
Setelah puas testing dan semuanya terbukti lancar di "localhost", ingat untuk melakukan kebalikan Step 3: mengubah kembali .env.production ke domain milik komdigi (misal `https://api.chatbot.domain.go.id`) sebelum file ini di-copy ke server VPS asli Anda!
