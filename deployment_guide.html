ℹ

Kenapa MongoDB lokal Docker? Lebih hemat biaya (tidak perlu MongoDB Atlas), data tersimpan di volume Docker sehingga persist walau container restart, dan tidak ada biaya per-request ke cloud.

Download GCP Service Account Key

💻 Laptop
  1. Buka console.cloud.google.com
  2. Pilih project kada-491403 di dropdown atas
  3. Menu kiri → IAM & Admin → Service Accounts
  4. Klik service account yang dipakai → tab Keys
  5. Klik Add Key → Create new key → JSON → Create
  6. File JSON otomatis ter-download
  7. Rename file menjadi gcp-key.json
  8. Simpan di Desktop sementara
🔐
File ini sangat rahasia! Jangan upload ke GitHub, jangan share ke siapapun. Akan di-upload manual ke VPS di fase berikutnya.

Install Docker

🖥️ VPS
 Bash (VPS)
# Update sistem
sudo apt update && sudo apt upgrade -y

# Install Docker via script resmi
curl -fsSL https://get.docker.com | sh

# Tambahkan user ke group docker
sudo usermod -aG docker $USER

# Apply perubahan group TANPA logout
newgrp docker

# Verifikasi
docker --version
docker compose version

Output yang diharapkan: Docker version 27.x.x dan Docker Compose version v2.x.x

# Aktifkan Nginx
sudo systemctl enable --now nginx

# Verifikasi berjalan

Setup Nginx & SSL Certificate

🖥️ VPS
 Bash (VPS)
# Copy config Nginx dari project
sudo cp /home/ubuntu/bpt-chatbot/nginx.conf /etc/nginx/sites-available/bpt-chatbot

# Hapus config default, aktifkan config baru
sudo rm -f /etc/nginx/sites-enabled/default
sudo ln -s /etc/nginx/sites-available/bpt-chatbot /etc/nginx/sites-enabled/

# Test syntax Nginx
sudo nginx -t
# Harus: "syntax is ok" dan "test is successful"

sudo systemctl reload nginx

# Dapatkan SSL certificate gratis dari Let's Encrypt
# Ganti domain di bawah dengan domain kamu
sudo certbot --nginx \
  -d chatbot.komdigi.go.id \
  -d api.chatbot.komdigi.go.id

# Ikuti prompt: masukkan email → A → N
# Certbot otomatis update nginx.conf dengan SSL

sudo nginx -t && sudo systemctl reload nginx
✅
SSL dari Let's Encrypt berlaku 90 hari dan auto-renew oleh Certbot. Tidak perlu perpanjang manual.

Build & Start Semua Service

🐳 Docker
 Bash (VPS)
cd /home/ubuntu/bpt-chatbot

# Build image dan jalankan semua container
# -d = background (detached)
# --build = paksa rebuild image
docker compose up -d --build
⏳
Proses ini memakan waktu 10–25 menit! Backend perlu install EasyOCR, PyTorch, sentence-transformers, dan semua ML dependencies. Jangan interupsi proses ini.

Pantau progress build:

 Bash (VPS)
# Log semua service (Ctrl+C untuk stop pantau)
docker compose logs -f

# Atau backend saja
docker compose logs -f backend

Tunggu hingga muncul:

Fase 5 — Buat Akun Admin

Seed akun admin pertama ke MongoDB lokal Docker

16

Jalankan Seed Script di dalam Container Frontend

🐳 Docker

Script seedAdmin.mjs harus dijalankan di dalam container frontend karena membutuhkan package mongoose dan bcryptjs yang sudah terinstall di sana.

 Bash (VPS)
# Masuk ke dalam container frontend
docker exec -it bpt_frontend sh

# Di dalam container, jalankan seed script
# Ganti password MongoDB sesuai yang kamu tentukan
MONGODB_URI="mongodb://bpt_admin:xK9mPq2rLw8nV3z!@mongodb:27017/bpt_db?authSource=admin" \
  node scripts/seedAdmin.mjs

# Keluar dari container setelah selesai
exit

Output yang diharapkan:

 Output
Connected to MongoDB
Created admin: admin
Created admin: operator1
Seeding complete!
🔄
Script ini idempotent — jika dijalankan dua kali, tidak akan duplikat. Admin yang sudah ada akan dilewati.

Backup Database MongoDB

🐳 Docker
 Bash (VPS)
# Dump database dari container MongoDB
docker exec bpt_mongodb mongodump \
  --authenticationDatabase admin \
  -u bpt_admin \
  -p "xK9mPq2rLw8nV3z!" \
  --db bpt_db \
  --out /tmp/backup

# Copy hasil dump dari container ke VPS host
docker cp bpt_mongodb:/tmp/backup \
  /home/ubuntu/backups/bpt-$(date +%Y%m%d)

# Verifikasi backup
ls -la /home/ubuntu/backups/
💡
Tambahkan perintah backup ini ke cron job agar berjalan otomatis setiap hari:
crontab -e → tambah: 0 2 * * * /home/ubuntu/backup.sh

Seed Script Error: "Cannot find module"

Jika file scripts/seedAdmin.mjs tidak ada di dalam container:

 Bash (VPS)
# Copy script ke dalam container secara manual
docker cp \
  /home/ubuntu/bpt-chatbot/bpt-komdigi-chatbot/scripts/seedAdmin.mjs \
  bpt_frontend:/app/scripts/seedAdmin.mjs

# Jalankan ulang seed
docker exec -it bpt_frontend sh
MONGODB_URI="mongodb://bpt_admin:PASSWORD@mongodb:27017/bpt_db?authSource=admin" \
  node scripts/seedAdmin.mjs
exit
!

Nginx 502 Bad Gateway

 Bash (VPS)
# Cek apakah container berjalan
docker compose ps

# Jika ada yang tidak running, start ulang
docker compose start frontend backend

# Cek port sudah listening
ss -tlnp | grep -E '3000|8000'
!

Login Admin Gagal / Password Salah

Jika lupa password atau ingin reset akun admin:

 Bash (VPS)
# Hapus admin lama dari MongoDB
docker exec -it bpt_mongodb mongosh \
  --authenticationDatabase admin \
  -u bpt_admin -p "PASSWORD_MONGO"

use bpt_db
db.admins.deleteOne({ username: "admin" })
quit()

# Seed ulang
docker exec -it bpt_frontend sh
MONGODB_URI="mongodb://bpt_admin:PASSWORD_MONGO@mongodb:27017/bpt_db?authSource=admin" \
  node scripts/seedAdmin.mjs
exit
