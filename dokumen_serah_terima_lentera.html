Gambaran Umum & Arsitektur Sistem

Tujuan, lingkup, dan desain arsitektur keseluruhan

1.1 Tujuan Sistem

LENTERA (Layanan Talenta Digital Responsif AI) adalah sistem chatbot berbasis AI yang dikembangkan untuk Balai Pelatihan Talenta (BPT) Komdigi. Tujuan utama sistem ini adalah menyediakan layanan informasi publik yang akurat, konsisten, dan tersedia 24 jam sehari tanpa membutuhkan kehadiran staf. Sistem dirancang khusus untuk menjawab pertanyaan seputar program pelatihan, prosedur pendaftaran, biaya, dan informasi umum BPT Komdigi berdasarkan dokumen resmi yang diunggah oleh administrator.

Pendekatan yang digunakan adalah Retrieval-Augmented Generation (RAG) — model AI generatif diperkaya dengan basis pengetahuan dari dokumen resmi, sehingga setiap jawaban bersumber dari informasi yang terverifikasi dan dapat dilacak ke dokumen sumbernya.

Alur Proses (Flow Diagram)

Diagram alur detail semua proses utama sistem

2.1 Alur Pengguna Bertanya ke Chatbot (11 Langkah)

1

Pengguna buka halaman chatbot

Browser load chatbot.domain.go.id. ChatWidget rendered sebagai floating button kanan bawah. Session ID dibuat/diambil dari localStorage (session_{timestamp}_{random}).

2

Widget dibuka → Cek kuota

Frontend panggil GET /api/rag/quota → proxy ke GET /api/quota?session_id=… FastAPI → Redis query key rl:{session_id}. Tampilkan pill "X/10" di header widget.

3

Pengguna ketik & kirim pertanyaan

Frontend kirim POST /api/rag/chat dengan body: {"query":"...","session_id":"..."}. Proxy ke POST /api/chat FastAPI (internal Docker).

4

Rate Limit Check — Redis

Key: rl:{session_id}. Jika count >= 10 → HTTP 429 + ttl_seconds. Frontend render CountdownTimer. Jika masih dalam limit → INCR counter (EXPIRE 10800s saat pertama), lanjut.

5

Exact Cache Lookup — Redis

Fingerprint = SHA-256(JSON({query_normalized, data_version, COLLECTION, K, chunk_size, chunk_overlap, llm_model, temperature, embedding_model, project, location, contact_info})). Key: rag:cache:v1:{hash}. Hit → return langsung.

6

Semantic Cache Lookup — Qdrant + Redis

Embed query → search koleksi rag_query_cache dengan filter: data_version + doc_collection + llm_model + embedding_model. Ambil top-5, score ≥ 0.92, token coverage ≥ 0.65, entity match. Ambil nilai dari Redis key di payload Qdrant.

7

Retrieval Dokumen — Qdrant

Embed query → cosine similarity search di koleksi bpt_docs → top-3 chunks paling relevan beserta metadata (filename, doc_id). Gabung menjadi context string.

8

Generate Jawaban — Gemini 2.5 Flash

Prompt template: "Anda adalah Asisten Resmi BPT Komdigi. Gunakan HANYA referensi berikut. Jika tidak ada → kalimat fallback." Chain: PromptTemplate | ChatVertexAI. Temperature 0.2.

9

Deteksi Fallback & Sumber

Cek marker fallback dalam jawaban. Jika fallback → is_fallback=true, sources=[]. Jika ada jawaban → sources=daftar nama file dokumen yang diretrieve.

10

Simpan & Cache Hasil

(a) MongoDB insert ke chats: session_id, query, answer, sources, timestamp WIB. (b) Redis SETEX exact cache 3600s. (c) Qdrant upsert ke rag_query_cache. (d) Invalidate top-questions Redis cache.

11

Frontend Render Response

Fallback → error card merah (warning icon + link WA + link Ticketing). Jawaban normal → render via Md.jsx (markdown). Update quota pill. Unread counter jika widget tertutup.

2.2 Alur Upload Dokumen Admin (8 Langkah)

1

Admin drag-drop atau pilih file

Validasi ekstensi & ukuran di frontend. Format: .pdf .docx .doc .csv .xlsx .png .jpg .jpeg .txt — Maks 15MB.

2

POST /api/rag/upload (Next.js Proxy)

multipart/form-data field file. Proxy ke POST /api/upload FastAPI.

3

Backend validasi & simpan ke temp file

Validasi ekstensi ulang. Save ke tempfile.NamedTemporaryFile dengan suffix ekstensi. Diberi prefix timestamp: {YYYYMMDDHHmmss}_{filename}.

4

Ekstraksi teks sesuai format

PDF → PyPDFLoader. DOCX → Docx2txtLoader. DOC → UnstructuredWordDocumentLoader. CSV → CSVLoader(utf-8). XLSX → pandas semua sheet → to_csv. PNG/JPG/JPEG → EasyOCR Reader singleton (id+en, gpu=False). TXT → TextLoader(utf-8).

5

Chunking — RecursiveCharacterTextSplitter

chunk_size=1000, chunk_overlap=100. Set metadata setiap chunk: doc_id (UUID baru) + filename.

6

Embedding & Store ke Qdrant

VertexAIEmbeddings(text-embedding-005) → vector 768-dim per chunk. QdrantVectorStore.add_documents() ke koleksi bpt_docs. Jarak: Cosine.

7

Catat metadata ke MongoDB

Insert ke documents: doc_id, filename, uploader, status="ready", uploaded_at (WIB ISO string).

8

Bump data version & cleanup

Redis INCR rag:data_version → cache lama otomatis invalid. Temp file dihapus. Return {"status":"success","doc_id":"...","filename":"..."}.

2.3 Alur Login Administrator (6 Langkah)

1

Akses /admin

Middleware Next.js cek cookie auth_token. Tidak ada JWT valid → tetap di /admin (LoginPage). Sudah ada → redirect ke /admin/dashboard.

2

Submit form login

POST /api/auth/login: {"username":"...","password":"..."}

3

Cari admin di MongoDB

Admin.findOne({username}). Tidak ditemukan → 401 (pesan generik tidak membedakan username/password salah).

4

Verifikasi password

bcrypt.compare(plainPassword, hashedPassword). Tidak cocok → 401.

5

Buat JWT & set cookie

SignJWT({username, role, avatar}).setExpiration('24h').sign(JWT_SECRET). Cookie: auth_token, HttpOnly=true, Secure=true, SameSite=Strict, maxAge=86400s, Path=/.

6

Redirect ke /admin/dashboard

Middleware verifikasi JWT di setiap request /admin/dashboard/* menggunakan jwtVerify(token, JWT_SECRET) dari library jose.

3.2 Spesifikasi Komponen

ChatWidget.jsx

  • Tipe"use client" — Client Component React
  • PropsisEmbedded (boolean, default false) — ubah posisi fixed→absolute untuk iframe mode

  • Error StatesisQuotaError (429): card merah + countdown + link WA + link Ticketing. isFallbackError: card merah + link WA + link Ticketing. Generic error: card merah biasa.

  • Propsadmin ({username, role, avatar, loginTime}), onLogout (callback)
  • Sections/Tabs"upload" (default), "manage", "analytics", "history"
  • Session TimerCountdown dari loginTime. SESSION_MS = 24h. Warna merah jika <5 menit. Auto-logout saat 0.
  • Upload FlowDrag&drop zone + file input. Validasi frontend (ext + size). Loading spinner. Result card. Auto-switch ke "manage" setelah 1s sukses.
  • Delete Flowwindow.confirm() → DELETE /api/rag/documents?id={doc_id} → refresh list.

5.2 Schema Pydantic (schemas.py)

schemas.py
class ChatRequest(BaseModel):
    query: str = Field(..., min_length=1, max_length=100)
    session_id: Optional[str] = "default_session"

class ChatResponse(BaseModel):
    answer: str                   # Jawaban dari Gemini (bisa markdown)
    sources: List[str]            # Nama file dokumen sumber (kosong jika fallback)
    session_id: str               # Echo dari request
    quota: Optional[dict] = None  # {limit, used, remaining, ttl}
    is_fallback: bool = False     # True jika info tidak ada di dokumen

class DocumentResponse(BaseModel):
    id: str; filename: str; status: str; uploaded_at: str

User Manual — Pengguna Chatbot

Panduan penggunaan chatbot untuk pengguna umum

7.1 Cara Menggunakan Chatbot LENTERA

Langkah 1 — Buka Halaman

Buka browser (Chrome/Firefox/Edge/Safari) dan akses https://chatbot.domain.go.id. Halaman menampilkan informasi BPT Komdigi dan tombol chat LENTERA di pojok kanan bawah.

Langkah 2 — Klik Tombol Chat

Klik tombol bulat dengan logo LENTERA. Jendela chat terbuka dengan pesan selamat datang. Tersedia 4 tombol pertanyaan cepat untuk memulai percakapan.

Langkah 3 — Ketik Pertanyaan

Ketik pertanyaan di kolom input (maks 100 karakter). Tekan Enter atau klik tombol kirim. Tunggu beberapa detik untuk respons AI.

Langkah 4 — Baca Jawaban

Jawaban ditampilkan dengan format markdown. Jika ada sumber dokumen, nama file ditampilkan di bawah. Jika informasi tidak ditemukan, muncul tombol link ke WhatsApp dan Ticketing System.

User Manual — Administrator

Panduan lengkap penggunaan panel admin

8.1 Login Panel Admin

  1. Buka https://chatbot.domain.go.id/admin
  2. Isi Username dan Password → klik Masuk
  3. Jika berhasil → otomatis masuk ke dashboard admin
⏱️

Session 24 Jam

Countdown timer di sudut kanan atas. Warna merah saat <5 menit. Sesi berakhir otomatis → redirect ke login.

8.2 Upload Dokumen Baru

  1. Klik 📤 Upload Dokumen di sidebar
  2. Drag file ke drop zone, atau klik "Pilih File"
  3. Format: PDF, DOC, DOCX, CSV, XLSX, PNG, JPG, TXT — Maks 15MB
  4. Tunggu proses selesai (spinner berputar)
  5. Kotak hijau = sukses  |  Kotak merah = gagal (baca pesan error)
💡

Tips Upload Dokumen

  • Untuk update info: hapus dokumen lama dulu, lalu upload baru
  • PDF dengan teks selectable lebih akurat dari PDF hasil scan
  • Gambar minimal 150 DPI untuk OCR yang baik

8.3 Kelola Dokumen

  1. Klik 📋 Kelola Dokumen
  2. Lihat daftar semua dokumen yang terindeks di Qdrant
  3. Hapus dokumen: klik ikon 🗑️ → konfirmasi → dokumen dihapus dari Qdrant & MongoDB
  4. Klik Refresh untuk muat ulang daftar

8.4 Analitik & Riwayat

Analytics (📊): Top pertanyaan dengan semantic clustering. Tampilkan pertanyaan paling sering beserta variannya. Gunakan untuk menentukan dokumen apa yang perlu ditambahkan.

Riwayat (💬): Semua log percakapan dengan filter pencarian keyword. Timestamp dalam WIB. Fitur "Hapus Semua Riwayat" tersedia (tidak bisa diurungkan).

Keamanan & Panduan Pemeliharaan

Implementasi keamanan dan checklist operasional rutin

12.1 Matriks Keamanan

AspekStatusImplementasi
Auth Password✓bcrypt salt 10 — password tidak pernah disimpan plaintext
JWT Token✓HS256, HttpOnly, Secure, SameSite=Strict, 24h expiry
Route Protection✓Next.js Edge Middleware verifikasi JWT setiap request /admin/dashboard/*
Network Isolation✓MongoDB/Qdrant/Redis tidak di-expose ke host — Docker internal only
DB Authentication✓MongoDB: username/password. Redis: requirepass.
CORS Restriction✓ALLOWED_ORIGINS env var — whitelist domain frontend
HTTPS Enforcement✓Nginx redirect HTTP→HTTPS. Let's Encrypt TLS 1.2/1.3.
Rate Limiting✓10 req/3jam per session_id via Redis INCR
Input Validation✓Pydantic max_length=100 query; whitelist ekstensi upload
Secret Management✓Semua secrets di .env (tidak di Git); GCP key mount :ro
Container PrivilegePerlu ReviewContainer berjalan sebagai root (default Docker)

12.2 Checklist Pemeliharaan Rutin

Harian

  • docker compose ps — pastikan semua container running
  • docker compose logs backend --tail=50 — cek error log

Mingguan

  • Review top-questions di analytics — identifikasi kebutuhan dokumen baru
  • Backup MongoDB: docker exec bpt_mongodb mongodump -u bpt_admin -p PASSWORD --db bpt_db --out /tmp/backup

Bulanan

  • Update Docker images: docker compose pull && docker compose up -d
  • Review billing Google Vertex AI di Cloud Console
  • Verifikasi SSL: sudo certbot certificates

12.3 Perintah Operasional

Bash (VPS)
# Status semua container
docker compose ps

# Log realtime (Ctrl+C untuk stop)
docker compose logs -f backend

# Update kode & rebuild
git pull && docker compose up -d --build

# Masuk MongoDB shell
docker exec -it bpt_mongodb mongosh --authenticationDatabase admin -u bpt_admin -p PASSWORD

# Restart service tertentu
docker compose restart backend

# Cek disk usage
df -h && docker system df
